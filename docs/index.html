1 React Architecture

React is built on a declarative component model. Understanding how components, properties (props), and state interact is essential for passing the final test.

💡 The React Flow Formula

State goes down (via props), Actions trigger events that go up (calling callbacks). Effects are synchronized side effects that run in response to dependency adjustments.

Props vs. State

Props are immutable parameters passed from a parent component down to a child. They behave like function arguments. State is a component's local memory that is mutable and triggers a re-render when modified via its setter function.

Side Effects (`useEffect`)

Use `useEffect` only for actions that escape the React render loop (fetching data, setting timers, or subscribing to events). Always declare all variables accessed in the effect inside the dependency array to avoid stale values.

  • State initialization: Declare states using descriptive names, e.g. const [isLoaded, setIsLoaded] = useState(false);
  • Conditional render check: Check for empty/null lists before mapping, e.g. movies?.length === 0 ? <EmptyState /> : movies.map(...).
  • Resource cleanup: Return a cleanup function inside useEffect to remove listeners or abort pending fetches.

2 Redux Toolkit & Async Thunks

Redux Toolkit (RTK) simplifies global application state. You'll need to define a store, write slice reducers, write async actions (thunks) for HTTP requests, and dispatch them.

⚠️ Common Pitfall: State Mutation

Your original codebase attempted state.movies.push(action.payload) inside the postAMovie.fulfilled case. Because state.movies is an object containing categorized lists, you must index into the target category list first: state.movies[category].push(movie).

Here is how to structure your movieSlice.js containing a GET request and a POST request:

src/slices/movieSlice.js 
import { createAsyncThunk, createSlice } from '@reduxjs/toolkit';
import { movieApi } from '../constants/axios';
import { movieRequests } from '../constants/requests';

// 1. Initial State matching your backend API's response layout
const initialState = {
  movies: {
    "Trending": [],
    "Comedy": []
  },
  status: 'idle', // 'idle' | 'loading' | 'succeeded' | 'failed'
  error: null
};

// 2. Async Thunk for GET request
export const getMovies = createAsyncThunk('movies/fetchMovies', async () => {
  const response = await movieApi.get(movieRequests.fetchAllMovies);
  return response.data.movies; 
});

// 3. Async Thunk for POST request (Adds a movie)
export const postAMovie = createAsyncThunk('movies/postMovie', async (movieData) => {
  const response = await movieApi.post(movieRequests.postMovie, movieData);
  return response.data.movie || movieData; 
});

// 4. Creating the slice
const moviesSlice = createSlice({
  name: 'movies',
  initialState,
  reducers: {},
  extraReducers: (builder) => {
    builder
      // Fetch Movies Cases
      .addCase(getMovies.pending, (state) => {
        state.status = 'loading';
      })
      .addCase(getMovies.fulfilled, (state, action) => {
        state.status = 'succeeded';
        state.movies = action.payload;
      })
      .addCase(getMovies.rejected, (state, action) => {
        state.status = 'failed';
        state.error = action.error.message;
      })
      // Post Movie Cases - FIXED to append into correct categorized array
      .addCase(postAMovie.fulfilled, (state, action) => {
        state.status = 'succeeded';
        const movie = action.payload;
        const category = movie.type || 'Trending';
        if (state.movies[category]) {
          state.movies[category].push(movie);
        } else {
          state.movies[category] = [movie];
        }
      });
  }
});

export default moviesSlice.reducer;

// Selectors to fetch values in your components
export const selectAllMovies = (state) => state.movies.movies;
export const getMoviesStatus = (state) => state.movies.status;
export const getMoviesError = (state) => state.movies.error;

Connect your page or component to dispatch and read from the Redux store:

src/pages/HomePage.jsx 
import React, { useEffect } from 'react';
import { useDispatch, useSelector } from 'react-redux';
import { getMovies, selectAllMovies, getMoviesStatus, postAMovie } from '../slices/movieSlice';

const HomePage = () => {
  const dispatch = useDispatch();
  const movies = useSelector(selectAllMovies);
  const status = useSelector(getMoviesStatus);

  // 1. Fetch data on mount if state is idle
  useEffect(() => {
    if (status === 'idle') {
      dispatch(getMovies());
    }
  }, [status, dispatch]);

  // 2. Handle POST request on form submission
  const handleSubmit = (event) => {
    event.preventDefault();
    const formData = new FormData(event.target);
    const movieData = Object.fromEntries(formData.entries());
    
    // Dispatch the async thunk
    dispatch(postAMovie(movieData));
  };

  return (
    <div>
      <form onSubmit={handleSubmit}>
        <input type="text" name="title" placeholder="Title" required />
        <input type="text" name="type" placeholder="Category" required />
        <button type="submit">Add Movie</button>
      </form>
      
      {status === 'loading' && <p>Loading movies...</p>}
      
      {status === 'succeeded' && Object.keys(movies).map(category => (
        <div key={category}>
          <h3>{category}</h3>
          {movies[category].map(m => <p key={m.movie_id}>{m.title}</p>)}
        </div>
      ))}
    </div>
  );
};

3 Custom Hooks

Custom hooks let you extract component logic into reusable functions. A highly common assignment asks you to write a useFetchData hook with standard properties.

🎯 The Custom Hook Contract

Custom hooks must start with the prefix use. They must return state variables (data, loading, error) and hook dependencies (like url) to allow React to track reactivity.

This hook accepts a endpoint/URL string and returns the API state:

src/hooks/useFetchData.js 
import { useEffect, useState } from "react";
import { movieApi } from "../constants/axios";

export const useFetchData = (url) => {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState("");

  useEffect(() => {
    // Use abort controller to cancel cleanup fetches if component unmounts
    const controller = new AbortController();

    const fetchData = async () => {
      setLoading(true);
      setError("");
      try {
        const response = await movieApi.get(url, {
          signal: controller.signal
        });
        setData(response.data);
      } catch (err) {
        if (err.name !== 'CanceledError') {
          setError(err.message || "Something went wrong");
        }
      } finally {
        setLoading(false);
      }
    };

    if (url) {
      fetchData();
    }

    return () => {
      controller.abort(); // clean up network connection
    };
  }, [url]); // Triggers only when URL changes

  return { data, loading, error };
};

To invoke the custom hook inside any React component:

src/components/MyComponent.jsx 
import React from 'react';

  );
};

4 Dynamic Form Generation

Dynamic forms render field configurations from API responses or static arrays. They are extremely common for validating frontend flexibility in exams.

⚙️ Prop Alignment Fix

Your DynamicFormPage.jsx passed formConfig={formConfig}, but the DynamicForm component destructured fields directly. We fixed it to pass fields={formConfig.fields} to avoid crashes.

Using third-party libraries is quick, but harder to customize. Here is the implementation using Formik & Yup:

src/components/DynamicForm.jsx 
import React from 'react';
import { Formik, Form, Field, ErrorMessage } from 'formik';
import * as Yup from 'yup';

const DynamicForm = ({ fields, onSubmit }) => {
  // 1. Generate dynamic Yup schemas
  const validationSchema = Yup.object().shape(
    fields.reduce((acc, field) => {
      if (field.type === 'email') {
        acc[field.name] = Yup.string().email('Invalid email').required('Required');
      } else if (field.type === 'password') {
        acc[field.name] = Yup.string().min(8, 'Min 8 characters').required('Required');
      } else {
        acc[field.name] = Yup.string().required('Required');
      }
      return acc;
    }, {})
  );

  // 2. Generate initial empty states
  const initialValues = fields.reduce((acc, field) => {
    acc[field.name] = '';
    return acc;
  }, {});

  return (
    <Formik initialValues={initialValues} validationSchema={validationSchema} onSubmit={onSubmit}>
      {() => (
        <Form>
          {fields.map((field) => (
            <div key={field.name} style={{ marginBottom: '15px' }}>
              <label>{field.label}</label>
              <Field type={field.type} name={field.name} />
              <ErrorMessage name={field.name} component="div" style={{ color: 'red' }} />
            </div>
          ))}
          <button type="submit">Submit</button>
        </Form>
      )}
    </Formik>
  );
};

If Formik is not allowed on the exam, you can write dynamic forms using standard React hooks:

Plain React Dynamic Form Component 
import React, { useState } from 'react';

const DynamicFormVanilla = ({ fields, onSubmit }) => {
  // Initialize form values dynamically
  const [values, setValues] = useState(() => 
    fields.reduce((acc, field) => ({ ...acc, [field.name]: '' }), {})
  );
  const [errors, setErrors] = useState({});

  // Handle dynamic field changes
  const handleChange = (name, val) => {
    setValues(prev => ({ ...prev, [name]: val }));
  };

  // Perform simple frontend validations
  const handleFormSubmit = (e) => {
    e.preventDefault();
    const tempErrors = {};
    fields.forEach(field => {
      if (!values[field.name]) {
        tempErrors[field.name] = `${field.label} is required`;
      }
    });
    
    if (Object.keys(tempErrors).length === 0) {
      onSubmit(values);
    } else {
      setErrors(tempErrors);
    }
  };

  return (
    <form onSubmit={handleFormSubmit}>
      {fields.map((field) => (
        <div key={field.name} style={{ marginBottom: '12px' }}>
          <label style={{ display: 'block' }}>{field.label}</label>
          
          {/* Render appropriate field type */}
          {field.type === 'textarea' ? (
            <textarea 
              value={values[field.name]} 
              onChange={(e) => handleChange(field.name, e.target.value)}
            />
          ) : (
            <input 
              type={field.type} 
              value={values[field.name]} 
              onChange={(e) => handleChange(field.name, e.target.value)}
            />
          )}
          
          {errors[field.name] && <span style={{ color: 'red' }}>{errors[field.name]}</span>}
        </div>
      ))}
      <button type="submit">Submit</button>
    </form>
  );
};

5 Skeleton Loaders

Skeleton loaders offer a smooth interface by rendering blank shapes that pulse while content is being fetched. This improves user experience compared to default progress bars.

Many exams restrict external packages. Implementing a basic shimmer skeleton in pure CSS is straightforward:

    90deg,

  background-size: 200% 100%;

6 React Context API

The Context API is React's native solution for sharing global state without "prop drilling". Used alongside useReducer, it behaves similarly to Redux.

The Provider Structure

You need createContext() to declare the context, and a Provider component wrapping children that passes down the current state and action dispatcher.

The Reducer Logic

A reducer function (state, action) => newState evaluates action types and yields copies of updated state models, matching your project's Auth layout.

src/context/AppStateReducer.js 
export const AppStateReducer = (state, action) => {
  switch (action.type) {
    case "Login": {
      localStorage.setItem("user", JSON.stringify({ ...action.payload, isAuthenticated: true }));
      return {
        ...state,
        isAuthenticated: true,
        user: action.payload
      };
    }
    case "Logout": {
      localStorage.removeItem("user");
      return {
        isAuthenticated: false,
        user: null
      };
    }
    default:
      return state;
  }
};
src/context/AppStateProvider.jsx 
import React, { createContext, useReducer } from 'react';
import { AppStateReducer } from './AppStateReducer';

const INITIAL_STATE = {
  isAuthenticated: localStorage.getItem("user") ? true : false,
  user: JSON.parse(localStorage.getItem("user"))
};

// 1. Create the Context
export const AppStateContext = createContext(INITIAL_STATE);

// 2. Create the Provider
const AppStateProvider = ({ children }) => {
  const [state, dispatch] = useReducer(AppStateReducer, INITIAL_STATE);

  return (
    <AppStateContext.Provider value={{ appState: state, dispatch }}>
      {children}
    </AppStateContext.Provider>
  );
};

export default AppStateProvider;

const [isSubmitting, setIsSubmitting] = useState(false);

const handleAddData = async (payload) => {

    setIsSubmitting(false);
  }
};

  • Error: "Cannot read property 'map' of undefined"
    Ensure initial state lists are arrays (e.g. []) or use optional chaining: items?.map(...).
  • Error: "Too many re-renders. React limits the number of renders..."
    You're calling a setter function directly inside the render cycle. Wrap callback executions: onClick={() => handleAction()} instead of onClick={handleAction()}.
  • Error: "Actions must be plain objects. Use custom middleware for async actions."
    Ensure you dispatch the execution of the async thunk: dispatch(getMovies()) (including parentheses) instead of dispatch(getMovies).
