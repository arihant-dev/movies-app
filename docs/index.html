React & Redux Toolkit — Exam Reference

Concise patterns, full working examples, and common pitfalls. Every code block is copy-pasteable.

React Core

useState

Returns a state variable and a setter. The setter triggers a re-render. Never mutate state directly.

useState patterns
// Simple value
const [count, setCount] = useState(0);

// Object state — always spread previous state
const [form, setForm] = useState({ name: '', email: '' });
setForm(prev => ({ ...prev, name: 'John' }));

// Array state — never push, always spread
const [items, setItems] = useState([]);
setItems(prev => [...prev, newItem]);

useEffect

Runs side effects after render. The dependency array controls when it fires.

useEffect patterns
// Runs once on mount (empty array)
useEffect(() => {
  fetchData();
}, []);

// Runs when 'id' changes
useEffect(() => {
  fetchMovieById(id);
}, [id]);

// Cleanup (timers, subscriptions, abort controllers)
useEffect(() => {
  const timer = setTimeout(() => setShow(false), 3000);
  return () => clearTimeout(timer); // cleanup runs before next effect or unmount
}, []);
Infinite loop trap
If you call a setter inside useEffect without the correct dependency array, you get an infinite re-render loop. If you see "Too many re-renders", check your useEffect deps.

Conditional Rendering

Three patterns
// 1. Ternary — use when you have both branches
{loading ? <Skeleton /> : <MovieList movies={movies} />}

// 2. Logical AND — use when you only show something conditionally
{error && <p className="error">{error}</p>}

// 3. Early return — use in function body for guard clauses
if (loading) return <Spinner />;
if (error) return <p>Error: {error}</p>;
return <MovieList />;

Rendering Lists

map with key
// Always provide a unique key. Never use array index if list can reorder.
{movies.map(movie => (
  <div key={movie.movie_id}>
    <h3>{movie.title}</h3>
    <img src={movie.poster} alt={movie.title} />
  </div>
))}

Why Provider?
Without <Provider>, any call to useSelector or useDispatch in a child component throws "Could not find store in context". It must wrap everything that needs store access.

RTK Slices & Async Thunks

Component calls dispatch(thunk())
→
Thunk runs async HTTP call
→
extraReducers handles pending/fulfilled/rejected
→
State updates → Component re-renders

Full Slice (GET + POST)

src/slices/movieSlice.js
import { createAsyncThunk, createSlice } from '@reduxjs/toolkit';
import { movieApi } from '../constants/axios';
import { movieRequests } from '../constants/requests';

const initialState = {
  movies: { "Top Rated": [], "Trending": [], "Comedy": [] },
  status: 'idle',  // idle | loading | succeeded | failed
  error: null
};

// ── GET ──
export const getMovies = createAsyncThunk(
  'movies/fetchMovies',
  async () => {
    const res = await movieApi.get(movieRequests.fetchAllMovies);
    return res.data.movies; // whatever the API returns goes to action.payload
  }
);

// ── POST ──
export const postAMovie = createAsyncThunk(
  'movies/postMovie',
  async (movieData) => {
    const res = await movieApi.post(movieRequests.postMovie, movieData);
    return res.data.movie || movieData;
  }
);

const moviesSlice = createSlice({
  name: 'movies',
  initialState,
  reducers: {},  // synchronous reducers go here if needed
  extraReducers: (builder) => {
    builder
      // GET cases
      .addCase(getMovies.pending, (state) => {
        state.status = 'loading';
        state.error = null;
      })
      .addCase(getMovies.fulfilled, (state, action) => {
        state.status = 'succeeded';
        state.movies = action.payload;
      })
      .addCase(getMovies.rejected, (state, action) => {
        state.status = 'failed';
        state.error = action.error.message;
      })
      // POST cases
      .addCase(postAMovie.pending, (state) => {
        state.status = 'loading';
      })
      .addCase(postAMovie.fulfilled, (state, action) => {
        state.status = 'succeeded';
        const movie = action.payload;
        const cat = movie.type || 'Trending';
        if (state.movies[cat]) {
          state.movies[cat].push(movie);  // Immer makes this safe inside RTK
        } else {
          state.movies[cat] = [movie];
        }
      });
  }
});

export default moviesSlice.reducer;

// ── Selectors ──
export const selectAllMovies = (state) => state.movies.movies;
export const getMoviesStatus = (state) => state.movies.status;
export const getMoviesError  = (state) => state.movies.error;
Critical: state.movies is an object, not an array
Since state.movies is { "Trending": [], "Comedy": [] }, calling state.movies.push(movie) crashes — .push is an array method. You must index into the correct category first: state.movies[category].push(movie).

Component Integration (Redux)

Two hooks: useSelector reads state, useDispatch triggers actions.

Full HomePage example
import React, { useEffect } from 'react';
import { useDispatch, useSelector } from 'react-redux';
import { getMovies, selectAllMovies, getMoviesStatus, postAMovie } from '../slices/movieSlice';
import Row from '../components/Row';

const HomePage = () => {
  const dispatch = useDispatch();
  const movies  = useSelector(selectAllMovies);
  const status  = useSelector(getMoviesStatus);

  // Fetch on mount if idle
  useEffect(() => {
    if (status === 'idle') dispatch(getMovies());
  }, [status, dispatch]);

  // Form submission → POST
  const handleSubmit = (e) => {
    e.preventDefault();
    const data = Object.fromEntries(new FormData(e.target).entries());
    dispatch(postAMovie(data));  // dispatch the THUNK, not a selector
  };

  return (
    <div>
      {status === 'loading' && <p>Loading...</p>}

      <form onSubmit={handleSubmit}>
        <input name="title" placeholder="Title" required />
        <input name="type" placeholder="Category" required />
        <button type="submit">Add Movie</button>
      </form>

      {Object.keys(movies).map(cat => (
        <Row key={cat} title={cat} movies={movies[cat]} />
      ))}
    </div>
  );
};

DELETE & PUT Thunks

Same pattern as POST. Change the HTTP method and handle state removal or update accordingly.

DELETE thunk + reducer
// ── Thunk ──
export const deleteMovie = createAsyncThunk(
  'movies/deleteMovie',
  async ({ movieId, category }) => {
    await movieApi.delete(`/movies/${movieId}`);
    return { movieId, category }; // return identifiers to remove from state
  }
);

// ── Inside extraReducers ──
.addCase(deleteMovie.fulfilled, (state, action) => {
  const { movieId, category } = action.payload;
  state.movies[category] = state.movies[category].filter(
    m => m.movie_id !== movieId
  );
})
PUT (update) thunk + reducer
// ── Thunk ──
export const updateMovie = createAsyncThunk(
  'movies/updateMovie',
  async ({ movieId, updates }) => {
    const res = await movieApi.put(`/movies/${movieId}`, updates);
    return res.data.movie;
  }
);

// ── Inside extraReducers ──
.addCase(updateMovie.fulfilled, (state, action) => {
  const updated = action.payload;
  const cat = updated.type;
  const idx = state.movies[cat]?.findIndex(m => m.movie_id === updated.movie_id);
  if (idx !== -1) {
    state.movies[cat][idx] = updated;
  }
})

Custom Hook: useFetchData

Encapsulates GET request + loading/error state. Return signature matches the assignment requirement exactly.

src/hooks/useFetchData.js
import { useEffect, useState } from 'react';
import { movieApi } from '../constants/axios';

export const useFetchData = (url) => {
  const [data, setData]       = useState(null);
  const [loading, setLoading] = useState(false);
  const [error, setError]     = useState('');

  useEffect(() => {
    const controller = new AbortController();

    const fetchData = async () => {
      setLoading(true);
      setError('');
      try {
        const res = await movieApi.get(url, { signal: controller.signal });
        setData(res.data);
      } catch (err) {
        if (err.name !== 'CanceledError') {
          setError(err.message || 'Request failed');
        }
      } finally {
        setLoading(false);
      }
    };

    if (url) fetchData();
    return () => controller.abort();
  }, [url]);

  return { data, loading, error };
};

Build a Row Component (End-to-End)

This is the exact exam scenario: "Given an API endpoint and some CSS, create a Row component that fetches and displays data." Here's how to approach it step by step.

Step 1: Understand what you receive

  • An API endpoint, e.g. /movies?category=Trending
  • CSS classes already provided, e.g. .row, .row_posters, .row_poster
  • You need to create a component that fetches from the endpoint and renders the data

Step 2: The Component

Row.jsx — fetches its own data
import React from 'react';
import { useFetchData } from '../hooks/useFetchData';
import '../styles/Row.css';

const Row = ({ title, endpoint }) => {
  const { data, loading, error } = useFetchData(endpoint);

  if (loading) return <div className="row"><h2>{title}</h2><p>Loading...</p></div>;
  if (error)   return <div className="row"><h2>{title}</h2><p>Error: {error}</p></div>;

  const movies = data?.movies || [];

  return (
    <div className="row">
      <h2>{title}</h2>
      <div className="row_posters">
        {movies.length === 0
          ? <span>No movies found</span>
          : movies.map(movie => (
              <img
                key={movie.movie_id}
                className="row_poster"
                src={movie.poster}
                alt={movie.title}
              />
            ))
        }
      </div>
    </div>
  );
};

export default Row;

Dynamic Forms

Render form fields from a JSON/array schema. This is the exam scenario: "given a schema, render a form."

The Schema

Form config (could come from API)
const fields = [
  { name: 'firstName', label: 'First Name', type: 'text' },
  { name: 'email',     label: 'Email',      type: 'email' },
  { name: 'role',      label: 'Role',       type: 'select', options: ['Admin', 'User', 'Guest'] },
  { name: 'age',       label: 'Age',        type: 'number' },
  { name: 'password',  label: 'Password',   type: 'password' }
];
DynamicForm.jsx
import React, { useState } from 'react';

const DynamicForm = ({ fields, onSubmit }) => {
  const [values, setValues] = useState(() =>
    fields.reduce((acc, f) => ({ ...acc, [f.name]: '' }), {})
  );
  const [errors, setErrors] = useState({});

  const handleChange = (name, value) => {
    setValues(prev => ({ ...prev, [name]: value }));
    // Clear error on change
    if (errors[name]) setErrors(prev => ({ ...prev, [name]: '' }));
  };

  const validate = () => {
    const errs = {};
    fields.forEach(f => {
      if (!values[f.name]) errs[f.name] = `${f.label} is required`;
      if (f.type === 'email' && values[f.name] && !values[f.name].includes('@'))
        errs[f.name] = 'Invalid email';
    });
    return errs;
  };

  const handleSubmit = (e) => {
    e.preventDefault();
    const errs = validate();
    if (Object.keys(errs).length > 0) {
      setErrors(errs);
      return;
    }
    onSubmit(values);
  };

  const renderField = (f) => {
    switch (f.type) {
      case 'select':
        return (
          <select value={values[f.name]} onChange={e => handleChange(f.name, e.target.value)}>
            <option value="">Select...</option>
            {f.options.map(o => <option key={o} value={o}>{o}</option>)}
          </select>
        );
      case 'textarea':
        return <textarea value={values[f.name]} onChange={e => handleChange(f.name, e.target.value)} />;
      default:
        return <input type={f.type} value={values[f.name]} onChange={e => handleChange(f.name, e.target.value)} />;
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      {fields.map(f => (
        <div key={f.name} style={{ marginBottom: '1rem' }}>
          <label style={{ display: 'block', marginBottom: '.25rem' }}>{f.label}</label>
          {renderField(f)}
          {errors[f.name] && <span style={{ color: 'red', fontSize: '.85rem' }}>{errors[f.name]}</span>}
        </div>
      ))}
      <button type="submit">Submit</button>
    </form>
  );
};

export default DynamicForm;
Formik + Yup version
import { Formik, Form, Field, ErrorMessage } from 'formik';
import * as Yup from 'yup';

const DynamicForm = ({ fields, onSubmit }) => {
  const schema = Yup.object().shape(
    fields.reduce((acc, f) => {
      if (f.type === 'email')    acc[f.name] = Yup.string().email('Invalid').required('Required');
      else if (f.type === 'number') acc[f.name] = Yup.number().required('Required');
      else                        acc[f.name] = Yup.string().required('Required');
      return acc;
    }, {})
  );

  const initial = fields.reduce((acc, f) => ({ ...acc, [f.name]: '' }), {});

  return (
    <Formik initialValues={initial} validationSchema={schema} onSubmit={onSubmit}>
      <Form>
        {fields.map(f => (
          <div key={f.name} style={{ marginBottom: '1rem' }}>
            <label>{f.label}</label>
            <Field name={f.name} type={f.type} />
            <ErrorMessage name={f.name} component="div" style={{ color: 'red' }} />
          </div>
        ))}
        <button type="submit">Submit</button>
      </Form>
    </Formik>
  );
};

Context API & useReducer

Three files: the reducer, the provider, and a custom hook to consume it cleanly.

1. Reducer

AppStateReducer.js
export const AppStateReducer = (state, action) => {
  switch (action.type) {
    case 'Login':
      localStorage.setItem('user', JSON.stringify(action.payload));
      return { ...state, isAuthenticated: true, user: action.payload };
    case 'Logout':
      localStorage.removeItem('user');
      return { isAuthenticated: false, user: null };
    default:
      return state;
  }
};

2. Provider

AppStateProvider.jsx
import React, { createContext, useReducer } from 'react';
import { AppStateReducer } from './AppStateReducer';

const INITIAL_STATE = {
  isAuthenticated: !!localStorage.getItem('user'),
  user: JSON.parse(localStorage.getItem('user'))
};

export const AppStateContext = createContext(INITIAL_STATE);

const AppStateProvider = ({ children }) => {
  const [state, dispatch] = useReducer(AppStateReducer, INITIAL_STATE);
  return (
    <AppStateContext.Provider value={{ appState: state, dispatch }}>
      {children}
    </AppStateContext.Provider>
  );
};
export default AppStateProvider;

Event Handling Patterns

These are the patterns that trip people up in exams. Know the difference between calling a function and passing a reference.

onClick patterns
//  CORRECT: passing a function reference (no args)
<button onClick={handleClick}>Click</button>

//  CORRECT: passing args via arrow function
<button onClick={() => handleDelete(movie.id)}>Delete</button>

//  CORRECT: passing the event object
<button onClick={(e) => handleClick(e, movie)}>View</button>

//  WRONG: this CALLS the function immediately on every render
<button onClick={handleDelete(movie.id)}>Delete</button>
Form submission pattern
const handleSubmit = (e) => {
  e.preventDefault();  // always prevent default to stop page reload

  // Option A: read from FormData (works with uncontrolled inputs)
  const formData = new FormData(e.target);
  const data = Object.fromEntries(formData.entries());

  // Option B: read from state (controlled inputs)
  // const data = { title, author, type };

  dispatch(postAMovie(data));
  e.target.reset(); // clear the form
};

    alert('Saved!');
  } catch (err) {
    setError(err.response?.data?.message || err.message);
  } finally {
    setSubmitting(false);
  }
};
// ── Controlled: React state drives the input value ──
const [name, setName] = useState('');
<input value={name} onChange={e => setName(e.target.value)} />

// ── Uncontrolled: Read values on submit via FormData ──
const handleSubmit = (e) => {
  e.preventDefault();
  const data = Object.fromEntries(new FormData(e.target));
  console.log(data.name); // reads from the input's name attribute
};
<form onSubmit={handleSubmit}>
  <input name="name" />
  <button>Submit</button>
</form>

Common Errors & Fixes

Errors you will encounter. Memorize the fix for each.

Cause: You're calling .map() on something that hasn't loaded yet or is undefined.

//  Crashes if movies is undefined
{movies.map(m => ...)}

//  Fix: optional chaining + fallback
{(movies || []).map(m => ...)}
// or
{movies?.map(m => ...) || <p>No data</p>}

Cause: You're calling a state setter during render, not inside an event handler or effect.

//  WRONG: calls setCount on every render → infinite loop
<button onClick={setCount(count + 1)}>

//  FIX: wrap in arrow function
<button onClick={() => setCount(count + 1)}>

Cause: You're passing the thunk function itself instead of calling it.

//  WRONG: passing the function reference
dispatch(getMovies);

//  FIX: call it with parentheses
dispatch(getMovies());

Cause: Missing key prop on mapped elements. React needs it to track list items.

//  WRONG
{items.map(item => <li>{item.name}</li>)}

//  FIX: use a unique identifier, NOT the index if list can change
{items.map(item => <li key={item.id}>{item.name}</li>)}

Cause: Closures capture the state value at the time the function was created.

//  Problem: count is always 0 inside the interval
useEffect(() => {
  const id = setInterval(() => setCount(count + 1), 1000);
  return () => clearInterval(id);
}, []);

//  Fix: use functional updater form
useEffect(() => {
  const id = setInterval(() => setCount(prev => prev + 1), 1000);
  return () => clearInterval(id);
}, []);

Cause: You're dispatching or setting state during the render phase of a parent/child.

//  WRONG: dispatch during render
const MyComponent = () => {
  dispatch(someAction()); // runs on every render!
  return <div />;
};

//  FIX: move it into useEffect
useEffect(() => {
  dispatch(someAction());
}, []);
